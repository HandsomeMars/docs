## Quick Start

### Create Transparent Background Video

```bash
curl --request POST \
  --url 'https://api.jogg.ai/open/v2/create_video_from_avatar' \
  --header 'x-api-key: YOUR_API_KEY' \
  --header 'Content-Type: application/json' \
  --data '{
    "script": "Welcome! This avatar has a transparent background.",
    "avatar_id": 127,
    "voice_id": "en-US-ChristopherNeural",
    "aspect_ratio": 0,
    "screen_style": 0,
    "transparent_background": true,
    "output_format": "mov"
  }'
```

<Check>
  Set `transparent_background: true` and `output_format: "mov"` for alpha channel support
</Check>

**Response:**

```json
{
  "code": 0,
  "msg": "success",
  "data": {
    "project_id": "vid_abc123"
  }
}
```

### Retrieve the Transparent Video

```bash
curl --request GET \
  --url 'https://api.jogg.ai/open/v2/avatar_video/vid_abc123' \
  --header 'x-api-key: YOUR_API_KEY'
```

**Response:**

```json
{
  "code": 0,
  "data": {
    "project_id": "vid_abc123",
    "status": "completed",
    "video_url": "https://res.jogg.ai/videos/vid_abc123.mov",
    "duration": 15,
    "has_alpha": true
  }
}
```

## Key Parameters

| Parameter | Value | Description |
|-----------|-------|-------------|
| `transparent_background` | `true` | Enable transparent background |
| `output_format` | `"mov"` or `"webm"` | Video format (MOV recommended for alpha) |
| `screen_style` | `0` | Must use full avatar style |

<Warning>
  Transparent background only works with `screen_style: 0` (full avatar). Styles 1 and 2 include background elements.
</Warning>

## Using Transparent Videos

### In Video Editors (After Effects, Premiere, Final Cut)

1. Import the MOV file
2. The alpha channel is automatically recognized
3. Place it over your background footage
4. Adjust position, scale, and timing as needed

### On Websites (HTML5 Video)

```html
<video autoplay loop muted playsinline>
  <source src="avatar_transparent.webm" type="video/webm">
  <source src="avatar_transparent.mov" type="video/quicktime">
</video>

<style>
video {
  position: absolute;
  width: 400px;
  height: auto;
  z-index: 10;
}
</style>
```

### In JavaScript/Canvas

```javascript
const video = document.createElement('video');
video.src = 'avatar_transparent.webm';

const canvas = document.getElementById('myCanvas');
const ctx = canvas.getContext('2d');

video.addEventListener('play', function() {
  function draw() {
    ctx.drawImage(video, 0, 0, canvas.width, canvas.height);
    requestAnimationFrame(draw);
  }
  draw();
});
```

## Best Practices

**Format Selection**
- Use MOV for video editing and post-production
- Use WebM for web applications and browsers
- Consider file size for your use case
- Test playback on target platform

**Avatar Selection**
- Choose avatars with clear edges
- Avoid avatars with complex hair (may have edge artifacts)
- Test different avatars for best results

**Technical Tips**
- Disable captions (`caption: false`) for clean overlay
- Use appropriate aspect ratios for your target
- Consider video duration vs. file size
- Cache videos for repeated use

**Post-Processing**
- You can further edit the transparent video
- Add effects, filters, or color correction
- Combine with other video elements
- Export in your desired final format

<Tip>
  For web use, provide both WebM and MOV formats as fallbacks for maximum browser compatibility.
</Tip>

## Limitations

<AccordionGroup>
  <Accordion title="Screen Style Restrictions">
Only `screen_style: 0` (full avatar) supports transparent backgrounds.

Styles 1 and 2 include background elements that cannot be made transparent.
  </Accordion>
  
  <Accordion title="Larger File Sizes">
Transparent videos are larger than standard videos:
- MOV: 5-10x larger than MP4
- WebM: 2-3x larger than MP4

Plan for longer download/upload times.
  </Accordion>
  
  <Accordion title="Processing Time">
Transparent videos take longer to process:
- Standard: 2-5 minutes
- Transparent: 3-7 minutes

Be patient and use webhooks for notifications.
  </Accordion>
  
  <Accordion title="Browser Compatibility">
**WebM with alpha**: Supported in Chrome, Edge, and Firefox. Not supported in Safari (use MOV instead).

**MOV with alpha**: Supported in Safari and most video players. Limited browser support for direct playback.

  </Accordion>
</AccordionGroup>

## Common Issues

<AccordionGroup>
  <Accordion title="Background Not Transparent">
    **Problem**: Video has black or white background  
    **Solution**: Ensure `transparent_background: true` and `output_format: "mov"` are set
  </Accordion>
  
  <Accordion title="File Too Large">
    **Problem**: MOV file is hundreds of MB  
    **Solution**: Use WebM format for smaller size, or compress the video post-processing
  </Accordion>
  
  <Accordion title="Edge Artifacts">
    **Problem**: Jagged or pixelated edges around avatar  
    **Solution**: Try different avatars, or use anti-aliasing in post-processing
  </Accordion>
  
  <Accordion title="Browser Won't Play">
    **Problem**: Transparent video doesn't play in browser  
    **Solution**: Use WebM for browsers, MOV for video editors. Provide both formats.
  </Accordion>
</AccordionGroup>
