## Supported Formats

| Format | Max Size | Max Duration | Bitrate | Use Case |
|--------|----------|--------------|---------|----------|
| **MP3** | 20MB | 10 minutes | Up to 320kbps | Best compatibility |
| **WAV** | 20MB | 10 minutes | Uncompressed | Highest quality |
| **M4A** | 20MB | 10 minutes | AAC codec | Good quality, small size |

## Upload Audio

```bash
curl --request POST \
  --url 'https://api.jogg.ai/open/v2/upload/asset' \
  --header 'x-api-key: YOUR_API_KEY' \
  --header 'Content-Type: multipart/form-data' \
  --form 'file=@/path/to/audio.mp3' \
  --form 'asset_type=audio' \
  --form 'name=Background Music'
```

**Response includes**:
```json
{
  "asset_id": "aud_ghi789",
  "duration": 120
}
```

## Recommended Settings

**For Background Music**
- Format: MP3
- Bitrate: 192-256 kbps
- Sample Rate: 44.1 kHz
- Channels: Stereo

**For Voice/Narration**
- Format: MP3 or M4A
- Bitrate: 128-192 kbps
- Sample Rate: 44.1 kHz
- Channels: Mono (saves space)

**Optimization**
```bash
# Using FFmpeg
# Convert to MP3 at 192kbps
ffmpeg -i input.wav -codec:a libmp3lame -b:a 192k output.mp3

# Normalize audio levels
ffmpeg -i input.mp3 -filter:a loudnorm output.mp3
```

## Best Practices

**Quality**
- Use at least 192 kbps for music
- Use 128 kbps for voice (sufficient)
- Normalize audio levels
- Remove silence at start/end

**File Size**
- Use MP3 for best size/quality ratio
- Use WAV only for master quality
- Compress long files
- Split long audio if needed

**Compatibility**
- MP3 has universal support
- M4A/AAC is modern and efficient
- WAV is large but lossless

## Common Issues

**Audio Too Quiet/Loud**
- Normalize audio levels before upload
- Use audio editing software
- Target -16 LUFS for music
- Target -18 LUFS for voice

**File Too Large**
- Convert WAV to MP3
- Reduce bitrate (192 â†’ 128 kbps)
- Trim unnecessary silence
- Split into segments

**Quality Loss**
- Don't use bitrate under 128 kbps
- Avoid multiple re-encodes
- Start with high-quality source
- Use appropriate format for use case
