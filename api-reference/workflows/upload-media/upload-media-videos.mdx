## Supported Formats

| Format | Max Size | Max Duration | Codec | Use Case |
|--------|----------|--------------|-------|----------|
| **MP4** | 100MB | 5 minutes | H.264/H.265 | Best compatibility |
| **WebM** | 100MB | 5 minutes | VP8/VP9 | Web optimized |
| **MOV** | 100MB | 5 minutes | H.264 | High quality |

<Warning>Videos longer than 5 minutes will be rejected</Warning>

## Upload Video

```bash
curl --request POST \
  --url 'https://api.jogg.ai/open/v2/upload/asset' \
  --header 'x-api-key: YOUR_API_KEY' \
  --header 'Content-Type: multipart/form-data' \
  --form 'file=@/path/to/video.mp4' \
  --form 'asset_type=video' \
  --form 'name=Product Demo'
```

**Response includes**:
```json
{
  "asset_id": "vid_def456",
  "duration": 30,
  "width": 1920,
  "height": 1080
}
```

## Recommended Settings

**Resolution**
- 1920x1080 (1080p) - Standard
- 1280x720 (720p) - Smaller size
- 3840x2160 (4K) - Only if necessary

**Encoding**
- Codec: H.264 (best compatibility)
- Bitrate: 5-8 Mbps for 1080p
- Frame Rate: 24, 30, or 60 fps
- Audio: AAC, 192 kbps

**Optimization**
```bash
# Using FFmpeg
ffmpeg -i input.mp4 \
  -vcodec h264 \
  -crf 23 \
  -preset medium \
  -vf scale=1920:1080 \
  -acodec aac \
  -b:a 192k \
  output.mp4
```

## Common Issues

**File Too Large**
- Reduce resolution (4K â†’ 1080p)
- Lower bitrate (use CRF 23-28)
- Shorten duration or split video
- Use two-pass encoding

**Codec Not Supported**
- Re-encode to H.264
- Use MP4 container
- Ensure audio is AAC

**Quality Issues**
- Don't use CRF > 28
- Use appropriate bitrate for resolution
- Avoid double-encoding
