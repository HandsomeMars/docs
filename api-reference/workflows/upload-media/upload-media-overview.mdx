## Introduction

Upload your own media assets to JoggAI to use in video creation. Include branded images, custom backgrounds, product photos, and audio files in your videos.

## Quick Start

### Upload an Image

```bash
curl --request POST \
  --url 'https://api.jogg.ai/open/v2/upload/asset' \
  --header 'x-api-key: YOUR_API_KEY' \
  --header 'Content-Type: multipart/form-data' \
  --form 'file=@/path/to/your/image.jpg' \
  --form 'asset_type=image' \
  --form 'name=Product Photo'
```

**Response:**

```json
{
  "code": 0,
  "msg": "success",
  "data": {
    "asset_id": "img_abc123",
    "asset_url": "https://res.jogg.ai/assets/img_abc123.jpg",
    "asset_type": "image",
    "file_name": "image.jpg",
    "file_size": 245678,
    "width": 1920,
    "height": 1080,
    "uploaded_at": 1732806631
  }
}
```

<Check>
  Save the `asset_id` and `asset_url` to use in your video projects!
</Check>

## Using Uploaded Assets

Once uploaded, reference your assets in video creation by using the `asset_id`:

```bash
# Use in avatar video
curl --request POST \
  --url 'https://api.jogg.ai/open/v2/create_video_from_avatar' \
  --header 'x-api-key: YOUR_API_KEY' \
  --header 'Content-Type: application/json' \
  --data '{
    "script": "Check out our amazing product!",
    "avatar_id": 127,
    "voice_id": "en-US-ChristopherNeural",
    "background_image": "img_abc123"
  }'
```

<Tip>
  You can use uploaded assets as backgrounds, product images, background music, or custom audio.
</Tip>

## Request Parameters

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `file` | file | Yes | The media file to upload |
| `asset_type` | string | Yes | Type: `image`, `video`, or `audio` |
| `name` | string | Yes | Descriptive name for the asset |
| `description` | string | No | Additional details or tags |
| `folder` | string | No | Organize into folders |

## Best Practices

**Before Uploading**
- Compress files to reduce upload time
- Use appropriate formats for each media type
- Resize to target resolution
- Test files locally first

**Organization**
- Use descriptive file names
- Leverage `folder` parameter to organize
- Document asset IDs for reuse
- Maintain a local asset library

**Quality**
- Don't upload unnecessarily large files
- Balance quality and file size
- Use recommended formats
- Optimize for target platform

## Common Issues

<AccordionGroup>
  <Accordion title="File Too Large">
    **Error**: `File size exceeds limit`
    
    **Solutions**:
    - Compress the file before upload
    - For images: reduce resolution or quality
    - For videos: lower bitrate or resolution
    - For audio: reduce bitrate or convert to MP3
  </Accordion>
  
  <Accordion title="Unsupported Format">
    **Error**: `File format not supported`
    
    **Solutions**:
    - Convert to supported format
    - Check file extension matches actual format
    - Ensure file is not corrupted
  </Accordion>
  
  <Accordion title="Upload Failed">
    **Error**: `Upload failed`
    
    **Solutions**:
    - Check internet connection
    - Verify API key is valid
    - Ensure file path is correct
    - Try uploading again
  </Accordion>
</AccordionGroup>
